* Project Name & One-liner Expense tracker
* Problem Statement - users dont remember where there money went 
* Core Features - sign-in, crud expenses
* Out of Scope- we are not buildning splitvise type app
* Tech Stack — jwt auth for login, passlib for password hashing, postgress for database,SQLlite as orm, fastapi for differnet endpoint of RESTAPI, python for logic building, pytest for testing the various inputs,for forntend (html,css,js)
* Success Criteria - can a user register, login, add an expense, and see only their own data

├── userlogin = auth and pasword
